Opciones de hosting — cómo publicarlo

Estos son los tres caminos para publicar los archivos. No necesitas conocimientos técnicos para ninguno.

A
Google Drive + Vercel Recomendado
Gratis, sin límites, URLs permanentes y acceso desde cualquier dispositivo.
  1. Ve a vercel.com y crea una cuenta gratis con tu email de empresa
  2. Haz clic en "Add New Project" → "Deploy from your computer"
  3. Sube los 2 archivos HTML (formulario-representantes.html y dashboard.html)
  4. Vercel te dará dos URLs como: tu-empresa.vercel.app/formulario-representantes
  5. Comparte la URL del formulario al equipo, guarda la del dashboard para ti
B
GitHub Pages
Gratis, técnicamente simple con la interfaz web de GitHub.
  1. Crea una cuenta en github.com
  2. Crea un repositorio nuevo, llámalo "agenda-ventas"
  3. Sube los 2 archivos HTML desde la interfaz web (botón "Add file")
  4. Ve a Settings → Pages → Source: selecciona "main branch"
  5. Tu URL será: tu-usuario.github.io/agenda-ventas/dashboard.html
C
Netlify Drop
La opción más rápida: arrastra y suelta los archivos, en 1 minuto tienes una URL.
  1. Ve a app.netlify.com/drop (no necesitas cuenta)
  2. Crea una carpeta en tu computador con los 2 archivos HTML
  3. Arrastra esa carpeta al área de drop de Netlify
  4. En 30 segundos tendrás una URL como: abc123.netlify.app
  5. Nota: sin cuenta, el sitio expira en 24h — crea cuenta gratis para que sea permanente

Punto crítico a entender

⚠️

Los datos se guardan en el navegador de cada persona

Actualmente, el formulario guarda los datos en el almacenamiento local del navegador (localStorage). Esto significa que los datos del formulario no se sincronizan automáticamente con el dashboard si están en computadores diferentes. Para que el sistema funcione en red real (varios dispositivos viendo los mismos datos), necesitas implementar Fase 5: base de datos compartida.


Mientras tanto, una solución simple: el representante y el gerente usan el mismo computador o navegador, o el gerente exporta los datos manualmente. Cuando el equipo esté adoptando bien el hábito, podemos conectar una base de datos gratuita (como Firebase o Supabase) para sincronización real entre dispositivos.
